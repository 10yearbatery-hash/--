# PRD: 커플 갈등 중재 서비스 — 본심 (Bon-Sim)

> ⚠️ **절대 구현 금지 (범위 외):** 소셜 로그인, BM/결제, 푸시 알림, 프로필 편집, A↔B 직접 대화, 갈등 기록 히스토리 열람, 관리자 페이지, 다국어, 앱 설정 페이지

---

## 1. 서비스 정의

연인 간 갈등 상황을 AI 에이전트가 중재하여, 감정이 아닌 진심으로 화해할 수 있도록 돕는 서비스.

- **앱 이름:** 본심 (Bon-Sim)
- **슬로건:** "비난 뒤에 숨겨진 진심을 함께 찾아요"
- **서브 슬로건:** "슬픔이 아닌, 관계의 회복"

---

## 2. 기술 방향

- 웹앱(모바일 웹) 형태로 구현한다.
- 향후 네이티브 앱 전환을 전제하므로, 모바일 퍼스트 반응형으로 설계한다.
- 별도 앱 설치 없이 URL 접속만으로 사용 가능해야 한다.
- 링크 공유 기반 플로우이므로 모바일 브라우저에서 완벽히 동작해야 한다.

---

## 3. 페이지 목록

| 번호 | 페이지 | 설명 |
|------|--------|------|
| 1 | 홈 | 메인 화면. 화해 플로우 시작점 |
| 2 | 우리들의 약속 | 합의된 약속 기록이 누적되는 페이지 |
| 3 | 방 만들기 | 이름과 갈등 키워드를 입력하고 방 코드를 생성하는 페이지 |
| 4 | 연인에게 공유하기 | 채팅방 코드/링크를 연인에게 공유하는 페이지 |
| 5 | 채팅방 입장 | 공유 코드/링크를 통해 채팅방에 입장하는 페이지 |
| 6 | 채팅방 | AI 에이전트와 1:1 대화하는 페이지 (각자 별도 채팅방) |
| 7 | 결과 페이지 | AI가 생성한 화해 결과를 4단계 스텝으로 보여주는 페이지 |
| 8 | 로그인 | 약속 저장 시 로그인을 유도하는 페이지 |

## 4. 유저 플로우

```
[홈]
  └─ "방 만들기" 버튼 클릭
      └─ [방 만들기 페이지]
          └─ 내 이름 / 상대 이름 / 갈등 키워드 입력 → "입력 완료 / 코드 생성" 클릭
              └─ [연인에게 공유하기 페이지]
                  └─ 방 코드/링크 복사 및 공유
                  └─ 본인도 "채팅방 입장하기" 버튼 클릭
                      └─ [채팅방] (A 전용)
                          └─ AI 에이전트와 대화 진행
                          └─ 대화 완료 시 대기 또는 결과 생성

[연인이 홈에서 "코드 입력하기" 클릭 또는 공유 링크 클릭]
  └─ [채팅방 입장 페이지]
      └─ "채팅방 입장하기" 버튼 클릭
          └─ [채팅방] (B 전용)
              └─ AI 에이전트와 대화 진행
              └─ 대화 완료 시 대기 또는 결과 생성

[A, B 모두 대화 완료]
  └─ AI가 결과 생성
      └─ 채팅방에 "결과보러가기" 버튼 표시
          └─ [결과 페이지] — 4단계 스텝
              └─ Step 1: 상황 요약
              └─ Step 2: 서로의 진심
              └─ Step 3: AI 갈등 통역
              └─ Step 4: 앞으로의 약속
                  └─ "약속 저장하기" 클릭
                      └─ (비로그인 시) [로그인 페이지] → 저장 후 모달
                      └─ (로그인 시) 저장 완료 모달 표시
                          └─ "처음으로 돌아가기" → [홈]
                          └─ 저장된 약속은 [우리들의 약속 페이지]에 누적
```

## 5. 페이지별 기능 명세

### 5-1. 홈

- 상단: 햄버거 메뉴(좌) + 로그인 버튼(우)
- 중앙: 로고(두 원이 겹치는 이미지), 앱 이름 "본심", 슬로건 카드
- "방 만들기" CTA 버튼 (핑크, 크고 눈에 띄게)
- "코드 입력하기" 텍스트 링크 (B가 상대방 코드로 입장할 때 사용)

### 5-2. 우리들의 약속

- 로그인한 유저의 약속 기록 목록을 시간순으로 표시한다.
- 각 약속 항목에는 날짜와 약속 내용이 표시된다.

### 5-3. 방 만들기

- 안내 카드: "대화를 시작해요 — 두 분의 이름과 갈등 키워드를 알려주세요"
- 두 분의 이름 섹션: 내 이름 입력 / 상대방 이름 입력
- 갈등의 핵심 섹션: 갈등 키워드 입력 (placeholder: "예: 연락 빈도, 약속 시간, 질투")
- 안내 문구: "키워드가 구체적일수록 더 정확한 도움을 받을 수 있어요"
- "입력 완료 / 코드 생성 →" 버튼 클릭 시 방이 생성되고 6자리 코드 발급
- 코드 생성 후 연인에게 공유하기 페이지로 이동

### 5-4. 연인에게 공유하기

- 생성된 방의 6자리 고유 코드 표시
- "링크 복사하기" 버튼
- "나도 채팅방 입장하기" 버튼 → A의 채팅방으로 이동

### 5-5. 채팅방 입장 페이지

- B가 코드/링크를 통해 접속하는 페이지
- 방 생성자(A)가 입력한 갈등 키워드 요약 표시
- "채팅방 입장하기" 버튼 → B의 채팅방으로 이동

### 5-6. 채팅방

- 헤더: "본심 상담" + 현재 질문 단계 태그 + 방 코드
- A와 B는 각각 별도 채팅방에서 AI 에이전트와 1:1 대화
- A와 B는 서로의 채팅 내용을 볼 수 없음
- 입력창 placeholder: "솔직하게 털어놔 보세요..."
- AI 응답 중 타이핑 인디케이터(`...`) 표시
- AI 에이전트 대화 로직은 섹션 6을 따름

### 5-7. 결과 페이지 — 4단계 스텝

상단 프로그레스 바로 현재 단계 표시. 각 스텝 하단에 "다음 >" 버튼.

#### Step 1: 상황 요약
- 타이틀: "상황 요약 / 무슨 일이 있었나요?"
- 섹션 헤더: "지금 상황을 정리해 봤어요"
- AI가 A/B의 Q1 답변을 바탕으로 감정 배제한 상황 요약 텍스트 출력
- 하단 강조 카드: 상황의 핵심 한 줄 ("두 분 모두 서로를 소중히 여기기 때문에 더 아팠던 거예요" 등)

#### Step 2: 서로의 진심
- 타이틀: "서로의 진심 / 서로의 속마음을 확인해요"
- 탭 전환: **[○○○의 입장]** / [○○의 입장] (이름 기반)
- 각 탭의 3단 구조:
  - ❤️ **가장 속상했던 부분** (Q2 기반) — 상대방 행동/말로 인한 상처
  - 💛 **정말로 바랐던 것** (Q3 기반) — 비난 뒤 숨은 진짜 니즈
  - 🤍 **상대에 대한 이해** (Q4 기반) — 상대 입장을 이해하려는 시도
- AI는 비난을 필터링하고 억울함과 진심만 남겨 출력

#### Step 3: AI 갈등 통역
- 타이틀: "AI 갈등 통역 / 본심으로 번역해드릴게요"
- AI가 제3자 입장에서 두 사람의 억울함이 결국 서로를 향한 애정에서 비롯되었음을 공감적으로 서술
- 출력 예시: "○○○님이 ○○에게 표현했을 때, 그 안에는 사실 '○'는 간절한 바람이 있었이요. 이건 비난이 아니라 관계에 대한 기대였어요. ○○님 역시 그 기대를 저버리고 싶었던 게 아니라, 자신도 최선을 다했지만 표현 방식이 달랐던 거예요."
- 하단 강조 카드: "두 분은 지금 멀어진 게 아니라, 서로를 향한 마음이 너무 커서 조금 엉켜 있는 거예요."

#### Step 4: 앞으로의 약속
- 타이틀: "앞으로의 약속 / 함께 지켜나갈 약속들"
- 섹션 헤더: "앞으로의 약속 💎 / 체크박스를 눌러 약속을 함께 해보세요"
- AI가 A/B 답변 기반으로 추천 약속 3~5개 생성
- 각 약속 항목: 번호 + 내용 + A/B 각각의 체크박스 (복수 선택 가능)
- "직접 약속 추가하기" 텍스트 입력 필드
- 하단 "약속 저장하기 💝" 버튼

#### 저장 완료 모달
- 💝 이모지 + "약속이 저장됐어요"
- "두 분의 소중한 약속이 기록됐어요. 로그인하면 언제든지 다시 볼 수 있어요."
- "처음으로 돌아가기" 버튼 (핑크)
- "닫기" 텍스트 링크

### 5-8. 로그인 페이지

- "약속 저장하기" 클릭 시, 비로그인 상태이면 이 페이지로 이동한다.
- 간단한 로그인 (이메일 Magic Link 방식)
- 로그인 완료 후 저장 처리 진행 → 저장 완료 모달 표시

## 6. AI 채팅 에이전트 명세

### 6-1. 전체 흐름

AI 에이전트는 아래 4개의 질문을 순서대로 진행한다. 각 질문은 유저의 답변이 해당 질문의 목적을 충분히 충족했다고 판단될 때 다음 질문으로 넘어간다. 답변이 불충분하면 추가 질문을 통해 더 깊이 들어간다.

### 6-2. 질문 목록 및 판단 기준

#### Q1: 상황 파악

- 질문: "먼저, 두 분이 부딪히게 된 '그 상황' 자체에 대해 듣고 싶어요. 감정을 조금 덜어내고, 어떤 일이 있었는지 있는 그대로 편하게 말씀해 주시겠어요?"
- 목적: 감정을 배제한 객관적 상황을 파악한다.
- 충족 조건: 언제, 어디서, 어떤 일이 있었는지 구체적인 상황이 파악되면 Q2로 넘어간다.
- 불충분한 경우: "조금 더 구체적으로 어떤 상황이었는지 알려주실 수 있을까요?" 등으로 추가 질문한다.

#### Q2: 핵심 상처/억울함 파악

- 질문: "그러셨군요, 그 상황이라면 정말 속상하셨겠어요. 그럼 그때 상대방의 어떤 말이나 행동 때문에 가장 상처를 받으셨나요? 특히 어떤 부분이 제일 억울하셨는지 여기에 다 털어놓아 보세요. 제가 다 들어드릴게요."
- 목적: 상대방의 어떤 행동/말이 상처였는지, 가장 억울한 포인트가 무엇인지 파악한다.
- 충족 조건: 상대방의 구체적 행동/말과 그로 인한 감정이 표현되면 Q3로 넘어간다.
- 불충분한 경우: "그 상황에서 가장 억울했던 순간이 있다면 어떤 순간이었나요?" 등으로 추가 질문한다.

#### Q3: 진짜 니즈(속마음) 파악

- 질문: "말씀을 듣고 보니 얼마나 억울하셨을지 짐작이 가네요. 그렇다면 화나고 서운했던 그 마음 이면에는, 사실 상대방이 '내게 이렇게 해줬으면 좋았을 텐데' 하고 바랐던 진짜 속마음이 있었을 거예요. 그건 무엇이었나요?"
- 목적: 비난 뒤에 숨어 있는 진짜 바람/니즈를 끌어낸다.
- 충족 조건: "상대방이 ~해줬으면 좋겠다"는 형태의 바람이 표현되면 Q4로 넘어간다.
- 불충분한 경우: "화가 난 건 결국 상대방에게 바라는 게 있었기 때문일 거예요. 진짜 원했던 건 뭐였을까요?" 등으로 추가 질문한다.

#### Q4: 상대방 입장 이해 유도

- 질문: "맞아요, 사실은 그런 마음이셨군요. 상대방도 이 진심을 꼭 알았으면 좋겠어요. 그럼 마지막으로 우리, 입장을 한 번만 바꿔서 생각해 볼까요? 비록 동의하기는 어렵겠지만, 상대방은 대체 왜 그런 행동이나 말을 했을까요? 그 사람에게도 나름의 '사정'이나 '이유'가 있었다면 무엇이었을지 한번 짐작해 볼까요?"
- 목적: 유저가 스스로 상대방의 입장을 이해하려는 시도를 하게 한다.
- 충족 조건: 상대방의 사정/이유에 대한 짐작이 표현되면 마무리 단계로 넘어간다.
- 불충분한 경우: "어렵겠지만, 상대방 입장에서 한 번만 생각해 본다면 그 사람도 이유가 있었을까요?" 등으로 추가 질문한다.

### 6-3. 마무리 단계

1. Q4 충족 후, "혹시 추가로 더 하고 싶은 말씀이 있으신가요?"를 물어본다.
2. 추가 발언을 듣거나 "없다"는 답변을 받으면 대화를 종료한다.
3. 대화 종료 후 분기 처리:
   - **IF** 상대방이 아직 대화 중 → "상대방이 아직 본심을 기록하고 있습니다. 조금만 기다려 주세요." 메시지 표시
   - **IF** 상대방도 대화 완료 → "두 분의 진심을 바탕으로 결과를 생성하고 있습니다..." 메시지 표시
4. 결과 생성 완료 후, 채팅방에 "결과보러가기 →" 버튼 표시
5. 버튼 클릭 시 결과 페이지(Step 1)로 이동

### 6-4. 에이전트 톤 & 태도

- 따뜻하고 공감적인 톤을 유지한다.
- 유저의 감정을 수용하되, 상대방을 비난하지 않는다.
- 자연스러운 대화 흐름을 유지한다. 기계적으로 질문을 던지지 않는다.
- 유저가 충분히 이야기할 수 있도록 공간을 준다.

## 7. 결과 페이지 출력 명세

결과는 **4단계 스텝 방식**으로 표시된다. 상단 프로그레스 바로 현재 단계를 시각화한다.

### Step 1: 상황 요약

- 데이터 소스: A와 B의 Q1 답변
- AI는 감정을 배제하고 갈등 상황을 객관적으로 요약한다.
- 출력 형식: 2~3문장 요약 + 핵심 한 줄 강조 카드

### Step 2: 서로의 진심

- 데이터 소스: A와 B의 Q2, Q3, Q4 답변
- UI: 탭(Tab) 형태로 A의 입장 / B의 입장을 전환하여 볼 수 있다.
- AI는 비난을 필터링하고 억울함과 진심만 남겨서 출력한다.
- 각 탭의 3단 구조:
  - ❤️ **가장 속상했던 부분** (Q2 기반)
  - 💛 **정말로 바랐던 것** (Q3 기반)
  - 🤍 **상대에 대한 이해** (Q4 기반)

### Step 3: AI 갈등 통역

- 데이터 소스: A와 B의 전체 답변
- AI가 제3자 입장에서 두 사람의 억울함이 결국 서로를 향한 애정에서 비롯되었음을 공감적으로 서술한다.
- 하단 강조 카드로 핵심 메시지 강조

### Step 4: 앞으로의 약속

- AI가 양측 답변을 기반으로 추천 약속 목록 생성 (3~5개)
- 각 약속에 A/B 각자가 체크박스로 동의 표시 (복수 선택 가능)
- 직접 약속을 텍스트로 입력하여 추가 가능
- "약속 저장하기 💝" 버튼 클릭 시:
  - 비로그인 → 로그인 페이지로 이동
  - 로그인 → 저장 완료 모달 표시
- 저장된 약속은 "우리들의 약속" 페이지에 누적
